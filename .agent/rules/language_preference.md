# Regla: Idioma de Interacción

- **Idioma Principal**: Todas las respuestas al usuario, explicaciones y comunicaciones deben ser en **español**.
- **Documentación de Artefactos**: Los planes de implementación, walkthroughs y resúmenes de tareas deben redactarse en español para mantener la consistencia con el usuario.
- **Código y Comentarios**: El código seguirá usando estándares universales (inglés para variables y funciones), pero los comentarios explicativos nuevos deben ser en español si se requiere claridad para el equipo local.
- **Excepción**: Los términos técnicos o nombres de herramientas (como "Skill", "Agent", "Firestore") se mantendrán en su forma original para evitar confusiones, pero se explicarán en español.
